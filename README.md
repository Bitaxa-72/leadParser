# Promotion Collector

Небольшой модульный проект для сбора публичных контактов бизнесов по городам в базу для ручной проверки и рассылки. Проект не отправляет письма, а только собирает и дедуплицирует найденные публичные данные.

## Что уже умеет

- Собирает кандидатов из OpenStreetMap через Overpass API.
- Опционально заходит на сайты из карт и пытается найти email и соцсети.
- Поддерживает seed-сайты в конфиге города: можно добавить локальные каталоги, сайты со списками бизнесов или страницы сообществ.
- Фильтрует записи по признакам русской/экспатской направленности.
- Классифицирует тип бизнеса по ключевым словам из конфига.
- Хранит технический файл `records.json`, чтобы не добавлять дубли.
- Ведет XLSX-файл по каждому городу и добавляет новый лист на каждую итерацию сбора.
- Держит данные разных городов в отдельных папках `data/<city-slug>/`.

## Быстрый старт

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
python -m promotion_collector.cli collect --city nha-trang --limit 10
```

Посмотреть доступные города:

```powershell
python -m promotion_collector.cli list-cities
```

Запустить только карты, без захода на сайты:

```powershell
python -m promotion_collector.cli collect --city nha-trang --limit 10 --sources overpass --no-enrich-websites
```

## Результаты

Для Нячанга файлы появляются здесь:

- `data/nha-trang/records.json` - техническое хранилище всех уже найденных записей.
- `data/nha-trang/nha-trang_contacts.xlsx` - таблица для ручной работы. Каждая новая итерация создает отдельный лист с датой запуска.

Столбцы XLSX:

- название бизнеса
- тип бизнеса
- город
- сайт
- почта
- Telegram
- VK
- Instagram
- Facebook
- WhatsApp
- YouTube
- другая соцсеть
- телефон
- адрес
- ссылка на источник данных
- название источника
- дата сбора

## Добавление города

Создайте файл `config/cities/<slug>.json`. Минимальный пример:

```json
{
  "slug": "example-city",
  "name": "Example City",
  "country": "Country",
  "bbox": [10.0, 20.0, 11.0, 21.0],
  "language_keywords": ["рус", "russian", "expat"],
  "business_type_keywords": {
    "Ресторан / кафе": ["restaurant", "cafe", "ресторан", "кафе"],
    "Сервис / услуги": ["service", "agency", "услуги"]
  },
  "seed_urls": [],
  "overpass_extra_filters": [
    {"tag": "name", "pattern": "russian|рус|expat"},
    {"tag": "website", "pattern": "\\.ru|vk\\.com|t\\.me"}
  ]
}
```

`bbox` задается как `[south, west, north, east]`.

## Ответственное использование

Скрипт рассчитан на публичные бизнес-контакты и ручную проверку. Перед рассылкой стоит проверить актуальность контактов, убрать личные адреса, соблюдать правила площадок-источников и местные требования к коммерческим сообщениям.
